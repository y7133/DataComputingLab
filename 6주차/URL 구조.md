<h3>프로토콜</h3>

- 위의 이미지에서 프로토콜은 'http'

- 대표적인 프로토콜은, http와 https

<h4>Http</h4>

![http](http.png)

- 웹 상에서 정보, 주로 html문서를 주고 받을 수 있는 프로토콜이며, 일반적인 웹 사이트에서 가장 많이 쓰임.

- 해당 프로토콜을 통해 클라이언트와 서버 사이에 이루어지는 요청과 응답을 주고 받을 수 있음

<h4>Https</h4>

![https](https.png)

- http의 보안이 강화된 버전으로,통신에서 주고 받는 요청과 응답 데이터를 일반 텍스트가 아닌 ssl이나 tcl을 통해 암호화 한 세션 데이터로 주고 받음
- 보안이 필요한 서비스(메일,은행,카드등)에서 주로 쓰이는 것을 확인 할 수 있음

<h3>도메인</h3>
<p>도메인은 'music.naver.com'
</p>
<p>도메인은 상위 도메인, 도메인 이름, 호스트명으로 이루어져 있음</p>
<p>도메인은 네트워크 상의 컴퓨터 주소를 의미하며 국가나 기관의 정보, 그리고 도메인 이름과 도메인에 속한 호스트 정보가 기입됨</p>
<p>각 정보는 점으로 구분할 수 있으며, 뒤에서부터 순차적으로 확인할 수 있음</p>
<h4>상위 도메인명</h4>

- 국가 구분
    - 캐나다 'ca'
    - 한국 'kr'
- 기관 구분
    - 영리목적 'com'
    - 비영리 목적 'org'
    
<h4> 도메인 명</h4>

- 인터넷상의 컴퓨터 주소는 숫자로 이루어진 IP주소로 표현되며, 이 숫자로 이루어진 주소를 알기 쉬운 영문으로 표현한 것이 도메인명

<h4>호스트명</h4>

- 호스트의 사전적 의미는 '인터넷에 연결된 지역의 이름이며 좀 더 쉽게는 도메인에 속한 컴퓨터 이름, 도메인의 하위개념으로 이해할 수 있음

- ex) news.naver.com 에서 도메인 이름은 'naver.com'이 되며, 호스트명은 naver.com 이라는 도메인에 속한 'news'가 됨 
호스트명은 'm.news.naver.com'에 붙은 'm.news'와 같이 2개 이상 사용할 수도 있으며, 'm.news.naver.com'은 new.naver.com 하위에 속한 추가 호스트 주소를 갖고있는 다른 url 로 해석할 수 잇음

<h4>파일 디렉토리 경로와 파일명</h4>

- 파일 도메인에 해당하는 웹서버를 기준으로, 제공하고자 하는 페이지가 속한 폴더명과 파일명이 나타난 경로를 나타냄

- 위의 이미지에서 'music.naver.com'이라는 도메인에 해당하는 웹서버의 listen 이라는 폴더에 저장되어있는 top100.nhn파일로 제공됨

<h4>파라미터</h4>

- 위의 이미지에서 domain과 duration이라는 파라키터 값을 각각 OVERSEA,1h로 설정하여 제공함

- <h4><b>가변적인 콘텐츠를 처리하기 위한 정보</b></h4>
    - 파라미터는 쿼리 스트링이라고도 하며, 정보에 따라서 페이지의 콘텐츠가 가변적일 때 많이 사용
    - 페이지는 하나이지만 검색어에 따라 다른 콘텐츠나 레이아웃으로 구성됨
    - 페이지 url에서 검색어와 같은 정보를 구분할 수 있도록 url뒤에 파라미터라는 매개변수를 추가하여 검색어 정보를 url에 함께 전달함
    - 파라미터는 url의 파일명 뒤 물음표 이후에 나열이 되며, 이름과 값으로 구성된 세트가 1개 이상 존재할 수 있음(세트는 엔드기호로 구분됨)

- URL을 구분하기 위한 정보
    - 한 페이지 url를 상황에 따라 구분하기 위한 용도의 추가정보로 파라미터를 사용할 수 있음
- 파일 디렉토리 및 파일명 역할을 하기 위한 정보
    - 파라미터는 동일한 페이지의 일부 정보를 구분하기 위한 용도외에도, 파일디렉토리 및 파일이름과 유사한 역할로도 쓰이는 경우가 있을 수 있음

<h3>플래그먼트</h3>

- 해당페이지에서 content영역을 지시하는 페이지

- 플래그먼트로 페이지의 특정 요소를 지시할 수 있음

- 콘텐츠를 이루는 요소 html 태그는 id라는 고유한 속성값을 가질 수 있음
- url 뒤에 해시태그를 붙인 요소의 id를 붙여서 확인하며, 해당 url페이지에서 해시태그로 지시한 id 요소를 스크롤 이동없이 확인 할 수 있음
- 여러 콘텐츠가 나열된 페이지의 특정 우치 정보를 사용자에게 함께 전달하고자 할 때 유용하게 쓰일 수 있음
